# Behavior Change Formula: SPARK × WANT × AGAIN × CAN

Behavior is a product of four forces. If any is zero or near-zero, behavior doesn't happen - even if the others are strong. Diagnose which force is failing before designing interventions.

---

## CAN (Capability)

The user is physically and cognitively able to perform the behavior.

**Diagnostic questions:**
- How many steps are required?
- How much thinking is required at each step?
- Does the user have all the information they need at the moment they need it?
- What would happen if we removed one step?
- What's the minimum viable version of this behavior?

**When CAN is the problem:**
- Users start but don't complete
- Users complete slowly or with visible hesitation
- High abandonment at form fields or decision points
- Support tickets asking "how do I do X?"

**CAN interventions:** Option Reduction, Smart Defaults, Pre-filling, Chunking, Specificity, Explicit Next Steps, Strategic Deadlines, Strategic Pauses

---

## WANT (Motivation)

The user has sufficient reason to act right now.

**Diagnostic questions:**
- What does the user gain immediately from this behavior?
- What does the user gain in the future?
- What do they lose by not doing it?
- What do people like them do?
- Does this connect to their identity or values?

**When WANT is the problem:**
- Users understand what to do but don't do it
- High awareness, low action
- Users say "I'll do it later" - and don't
- Behavior completes once but doesn't repeat

**WANT interventions:** Social Proof, Identity Framing, Personalization, Loss Aversion Framing, Progress Visibility, Goal Gradient, Anchoring, Priming, Meaning Addition, Implementation Intention Prompt

---

## SPARK (Trigger)

Something prompts the user to act at the right moment.

**Diagnostic questions:**
- When does the user encounter this decision?
- Is there a natural prompt in the current flow?
- Is the prompt arriving at a high-receptivity moment?
- Is the prompt competing with too many other prompts?
- Is there any prompt at all?

**When SPARK is the problem:**
- Motivated, capable users still don't act without reminders
- Behavior happens when someone is reminded but not otherwise
- Usage is reactive (help-desk driven) rather than proactive

**SPARK interventions:** Contextual Timing, Multiple Touchpoints, Ambient Reminders, Event-Based Triggers

---

## AGAIN (Repetition)

The behavior is reinforced enough to become a habit or sustained change.

**Diagnostic questions:**
- Is there a reward structure after the behavior?
- Can the behavior be connected to an existing routine?
- Is progress visible over time?
- Does the user have a reason to come back and repeat?

**When AGAIN is the problem:**
- User completes the behavior once, then stops
- Good Day-1 activation, poor Day-7 and Day-30 retention
- Behavior requires reminders every time; never becomes automatic

**AGAIN interventions:** Habit Stacking, Variable Reward, Celebration Moments, Streak Mechanics, Reinforcement Timing, Review Loops

---

## Diagnostic Flow

```
Does the user know they should do this?
→ No → SPARK problem (trigger missing)
→ Yes ↓

Is the behavior easy enough?
→ No → CAN problem (too much friction)
→ Yes ↓

Is the user sufficiently motivated right now?
→ No → WANT problem (motivation gap)
→ Yes ↓

Has the behavior become a habit?
→ No → AGAIN problem (no reinforcement)
→ Yes → Behavior is working - focus on scaling
```

---

## Force Interaction Effects

Forces multiply, not add. A weak WANT can be partially compensated by a very strong CAN (make it so easy they barely need motivation), but there are limits.

**Common patterns:**

| Pattern | Symptom | Fix |
|---------|---------|-----|
| High WANT + Low CAN | "I really want to but keep getting confused" | Fix friction first |
| High CAN + Low WANT | "It's easy but I don't care enough" | Fix motivation or reconsider value proposition |
| Low SPARK + High WANT/CAN | "I forget about it" | Fix triggers |
| Good SPARK/WANT/CAN + Low AGAIN | "I tried it once" | Fix reinforcement loops |

Sustainable behavior change usually requires all four forces to be at least adequate. Optimizing one while neglecting another produces inconsistent results.
